# Phiên bản

## 1. Mục tiêu tài liệu
- Ghi nhận các phiên bản đã triển khai.
- Theo dõi danh sách tính năng theo từng phiên bản.
- Làm nguồn tra cứu nhanh khi đối soát phạm vi release.

## 2. Nhật ký phiên bản đã triển khai

| Phiên bản | Ngày triển khai | Trạng thái | Phạm vi chính |
|---|---|---|---|
| `v0.1.0` | `2026-05-22` | `Đã triển khai` | Khung tài liệu kiến trúc, quy ước, cấu trúc dự án |

## 3. Danh sách tính năng theo phiên bản

### 3.1 `v0.1.0`
- Thiết lập bộ tài liệu nền tảng cho dự án.
- Chuẩn hóa cấu trúc response API (`success/message/data/meta`).
- Chuẩn hóa quy ước database (`created_at`, `updated_at`, `is_deleted`, `deleted_by`, `deleted_at`).
- Chuẩn hóa kiến trúc hệ thống theo mô hình monolith (`UI`, `Service API`, `Database`, `Redis Cache`, `Internet/External`).

## 4. Quy tắc cập nhật tài liệu phiên bản
- Mỗi lần triển khai phiên bản mới, bắt buộc thêm một mục mới trong bảng phiên bản.
- Mỗi phiên bản phải liệt kê rõ tính năng mới, tính năng thay đổi, tính năng loại bỏ (nếu có).
- Nếu có thay đổi phạm vi nghiệp vụ, cập nhật đồng thời ở tài liệu `Tổng quan nghiệp vụ`.
