## 2. Xác thực và phân quyền

- Mô hình phân quyền: <RBAC/ABAC/...>

## 3. Bảo mật dữ liệu
- Dữ liệu cần mã hóa khi lưu trữ.
- Dữ liệu cần bảo vệ khi truyền tải.
- Quy tắc ẩn/mask dữ liệu nhạy cảm trong log.

## 4. Bảo vệ API
- Rate limit.
- Kiểm tra input/validation.
- CORS: chỉ cho phép origin từ `FRONTEND_ORIGIN`, bật `credentials: true`.
- Callback token: dùng cookie JWT HttpOnly `google_change_token`, TTL 60 giây.
- Cookie callback Google: `NODE_ENV=production` dùng `SameSite=None; Secure`, các môi trường còn lại dùng `SameSite=Lax; Secure=false` để hỗ trợ local dev.
- One-time code: lưu mapping `code -> email` trong Redis TTL 60 giây và xóa ngay sau khi exchange thành công.
- Exchange token: response chỉ trả access token + Google user; refresh token lưu trong cookie HttpOnly `refresh_token`, TTL 7 ngày.
- Chống injection và các tấn công phổ biến.

## 5. Quản lý bí mật và khóa
- Lưu trữ secret qua biến môi trường/vault.
- Quy trình xoay vòng khóa/secret.

## 6. Giám sát và ứng phó sự cố
- Audit log bảo mật.
- Cảnh báo bất thường.
- Quy trình xử lý incident.
